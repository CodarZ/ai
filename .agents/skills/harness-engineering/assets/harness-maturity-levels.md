# Harness 成熟度自检（L0–L5）

> 用 6 级模型评估你的项目当前在哪一阶段，下一步该做什么。

---

## 总览

| Level  | 名称                | 关键特征                      | 典型团队               |
| ------ | ------------------- | ----------------------------- | ---------------------- |
| **L0** | 裸奔                | 直接 prompt LLM，无任何约束   | 个人玩具               |
| **L1** | 基础 Rule           | 有基础 Rule，靠人记           | 初创小团队             |
| **L2** | Rule + Skill        | 高频动作脚本化 + Scripts 闸机 | 中型团队 AI 改造期     |
| **L3** | 多 Agent + Workflow | 结构化调度 + 流程定义         | 大型团队 AI 转型       |
| **L4** | 完整 6 零件         | 全套 + dev-map + Pre-PR       | 美团/腾讯标杆          |
| **L5** | 自演进 Harness      | 错误自动沉淀为 Rule/Skill     | 行业领先（暂未规模化） |

---

## L0：裸奔

### 特征

- AI 用法：复制粘贴 Prompt
- 没有 .cursorrules / CLAUDE.md
- 没有项目级 Skill
- 不验证 AI 产出

### 痛点

- AI 反复犯同样的错
- 不同任务行为不一致
- 不知道 AI 改了什么
- 出问题找不到根因

### 应该做的

→ 直接读 `playbooks/new-project.md` 或 `ongoing-project.md`，开始建 L1。

---

## L1：基础 Rule

### 特征

- 有基础的 .cursorrules / CLAUDE.md
- 5-10 条 Rule
- 没有 Skill 或只有 1-2 个
- 测试靠手跑

### 自检题

- [ ] 你有项目级 always Rule 吗？
- [ ] AI 知道项目的命名规范吗？
- [ ] 有 SPEC 文档吗？

### 升 L2 的信号

- AI 在某些固定流程反复出错（应升级为 Skill）
- 团队加入第 2 个人
- 代码量超 5000 行

### 升 L2 的动作

1. 写 3-5 个高频 Skill
2. 建立基础 Scripts（编译 + 测试 + 测试数检查）
3. 引入基线对比机制

---

## L2：Rule + Skill + Scripts 闸机

### 特征

- 有 always Rule + 5-10 个 Skill
- 有 Scripts 闸机（基础三类检查）
- 有 SPEC 文档
- 测试有 CI 跑

### 自检题

- [ ] Scripts 跑过才能 commit / merge 吗？
- [ ] 有基线对比机制吗？
- [ ] 测试数检查在吗？
- [ ] AI 高频动作下沉为 Skill 了吗？

### 升 L3 的信号

- 单 Agent 在复杂任务中失稳
- 需要"专业判断"（如代码审查、测试设计）
- 团队规模 > 5

### 升 L3 的动作

1. 拆出 2-3 个 Sub Agent（如设计/实现/测试）
2. 定义初步 Workflow（流程定义 + 角色契约）
3. 引入模型分层（PM 用小、专业用大）

---

## L3：多 Agent + Workflow

### 特征

- 2-7 个 Sub Agent
- 有 Workflow 定义文件
- 有角色契约
- 模型分层配置

### 自检题

- [ ] 每个 Agent 职责清晰，不越界？
- [ ] 下游能阻塞上游而不是直接改吗？
- [ ] 流程定义有可校验脚本吗？
- [ ] PM Agent 只做路由吗？

### 升 L4 的信号

- 项目规模超 5 万行
- 多人协作时找不到代码入口
- 重复造轮子
- CR 速度跟不上 AI 出码速度

### 升 L4 的动作

1. 建 dev-map（开发导航地图）
2. 建任务看板
3. 引入 Pre-PR 机制
4. 引入高低阶模型互审、跨厂商对抗

---

## L4：完整 6 零件 + dev-map + Pre-PR

### 特征

- Rule / Skill / Sub Agent / Workflow / Scripts / MCP 都有
- dev-map + 任务看板
- Pre-PR 机制
- 4 块拼图齐全（约束流程 / 反馈 / 知识库 / 进化）

### 自检题

- [ ] 4 块拼图都有吗？
- [ ] 改代码前先查 dev-map 是文化吗？
- [ ] CR 聚焦业务语义而非找错字吗？
- [ ] 测试有 Human-in-the-loop SOP 吗？
- [ ] 有跨模型互审吗？

### 升 L5 的信号

- 想让 Harness 自我演进
- 错误能自动转为 Rule/Skill 的提案

### 升 L5 的动作

1. 建 Error-Driven 闭环自动化
2. 出错时自动 PR 提议加 Rule
3. AI 自我评估 Harness 哪里需要补

**注意**：L5 目前还没有规模化达成，是前沿。

---

## L5：自演进 Harness（前沿）

### 特征

- 错误自动沉淀为 Rule/Skill 提案
- Harness 本身可被 AI 优化
- 持续监控 Harness 健康度

### 当前状态

- 行业前沿，暂未规模化
- 部分团队在探索

---

## 4 块拼图自检（L3+ 必看）

四块拼图：约束流程 / 反馈 / 知识库 / 进化。

| 拼图         | L1        | L2           | L3         | L4                   |
| ------------ | --------- | ------------ | ---------- | -------------------- |
| **约束流程** | 基础 Rule | + Skill      | + Workflow | + MCP                |
| **反馈**     | 测试      | + Scripts    | + 基线对比 | + Pre-PR             |
| **知识库**   | SPEC      | + Skill 文档 | + 角色契约 | + dev-map / 任务看板 |
| **进化**     | （无）    | 手动加 Rule  | 复盘升级   | Error-Driven 闭环    |

## 各 Level 的典型 ROI

| Level   | 投入时间 | 收益              |
| ------- | -------- | ----------------- |
| L0 → L1 | 1-2 天   | 减少 30% 反复错误 |
| L1 → L2 | 1-2 周   | 减少 50% 完成幻觉 |
| L2 → L3 | 1 月     | 复杂任务可控      |
| L3 → L4 | 2-3 月   | 大规模团队可扩展  |
| L4 → L5 | 持续     | 自演进            |

**重要**：不要为了升 Level 而升 Level。每一步都应有明确**痛点**驱动。

---

## 反模式：跳级

| 跳级               | 风险                   |
| ------------------ | ---------------------- |
| L0 直接 L4         | 维护成本爆炸，没人能用 |
| 没痛点就升级       | 浪费资源               |
| 单纯模仿"大厂方案" | 不适合自己情况         |

**关键原则**：每一级都应该是**问题驱动**，不是"看起来应该升级了"。
